Executive Summary

SmartFlow is an enterprise workflow management and job tracking system built for Minapharm (pharmaceutical). It manages the full lifecycle of packaging/labeling change requests — from initial submission through multi-stage approval workflows, to final job completion and master file archival.

Request Submission
Packaging label update for a drug product
Review & Approval
Multi-department review and sign-off
Job Creation
Approved items become executable jobs
Workflow Execution
Sequential / parallel process functions with time tracking
Master File Management
Versioned document control and approval
Job Closure & Reporting
Analytics, delay tracking, and archival

Business Domain

ConceptDescription
RequestA change control request (e.g., "Update label for Product X"). Contains one or more Request Items.
Request ItemA single line item specifying a product, target, and workflow to apply.
JobAn executable unit of work created from an approved Request Item. Goes through a Workflow.
WorkflowA template defining the sequence of Processes needed to complete a type of Job. Bound to a Target.
ProcessA stage within a Workflow, owned by a Department (e.g., "Design Review", "Quality Approval").
Process FunctionAn individual task within a Process (e.g., "Review artwork"). Has owners, lead time, and inputs.
TransactionA runtime instance of a Process Function being executed for a specific Job. Tracks status and timing.
Product Master FileA versioned document (artwork, specification) associated with a product, with its own approval flow.
WorkspaceMulti-tenancy boundary — users and data are scoped to workspaces.

User Roles

Requester
Submits change requests with product details and change control documents.
Request Approver
Reviews and approves or rejects request items.
Function Owner
Executes assigned functions within a job (review, approve, design).
Workflow Admin
Manages workflow templates and process configurations.
System Admin
Manages users, roles, claims, workspaces, and lookups.
Manager
Views dashboards, workload analysis, and delay reports.

Timing & Workload Calculation

How Lead Time Works
Each ProcessFunction has a LeadTime (hours). TransactionTime calculates expected dates considering: function dependencies (sequential wait for predecessors), 8-hour workdays, official holidays (excluded), and the ApplyWorkLoad flag for distributing work across owners. The TimingService background job periodically recalculates these dates.
